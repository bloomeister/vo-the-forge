# Contributing

## Code Style

- Follow `.editorconfig`.
- C/C++: use tabs for indentation.
- Markdown: use spaces (2) for indentation.
- Keep changes minimal and focused.

## Documentation

- Prefer short, step-by-step instructions.
- Include paths and exact menu names where helpful.
- Keep examples copy/pasteable.

## Pull Requests

- Describe what changed and why.
- Include repro steps for bug fixes.
